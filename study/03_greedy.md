# 第3章：貪欲法

## 3-1. 貪欲法の考え方

貪欲法（グリーディ法）とは、**各ステップで「今最も良い選択」を繰り返す**手法です。
将来を見通さずに局所最適な選択をするだけで、全体最適に到達できるケースに使えます。

### 全探索・DP との比較

| | 全探索 | 貪欲法 | DP |
| --- | --- | --- | --- |
| 方針 | 全候補を試す | 毎回「最善」を選ぶ | 部分問題の最適解を積み上げる |
| 計算量 | 大きい | 小さい（ソート＋線形など） | 中程度 |
| 正しさ | 常に正しい | 問題の性質に依存 | 常に正しい |

### 貪欲法が成り立つ直感

「今の最善選択が、将来の選択肢の価値を下げない」ことが保証できるとき。
これを**交換論法**（3-4節）で確認する。

## 3-2. ソートして貪欲に選ぶ

最もよく出るパターン。**ある基準でソートして、先頭（または末尾）から順に条件を満たすものを選ぶ**。

### 例：価値の高い商品から順に袋に詰める

```cpp
int main() {
    ll n, W;
    cin >> n >> W;

    vector<pll> items(n); // {価値, 重さ}
    rep(i, n) cin >> items[i].first >> items[i].second;

    sort(rall(items)); // 価値の高い順

    ll total = 0, cap = W;
    for (auto [v, w] : items) {
        if (cap >= w) {
            total += v;
            cap -= w;
        }
    }
    cout << total << endl;
}
```

### ソート基準の選び方

| 問題のパターン | ソート基準 |
| --- | --- |
| 価値を最大化 | 価値の降順 |
| コストを最小化 | コストの昇順 |
| 締切に間に合わせる | 締切の昇順 |
| 区間の重なりを避ける | 終了時刻の昇順（→ 3-3節） |

ソート基準が正しいかは**交換論法**（3-4節）で確認する。

## 3-3. 区間スケジューリング

「複数の仕事（区間）があり、重ならないように最大数こなす」という典型問題。

### 戦略：終了時刻が早い順に選ぶ

```
仕事一覧：
  A: [1, 2)
  B: [1, 4)
  C: [3, 5)
  D: [4, 6)
  E: [5, 7)

終了時刻順：A(2), B(4), C(5), D(6), E(7)
  → A を選ぶ（終了時刻 2、現在の終了 = 2）
  → B：開始 1 < 2 → スキップ
  → C：開始 3 ≥ 2 → 選ぶ（現在の終了 = 5）
  → D：開始 4 < 5 → スキップ
  → E：開始 5 ≥ 5 → 選ぶ（現在の終了 = 7）
  結果：[A, C, E] = 3件（最大）
```

### 実装

```cpp
int main() {
    ll n;
    cin >> n;
    vector<pll> jobs(n); // {終了時刻, 開始時刻}
    rep(i, n) {
        ll s, e;
        cin >> s >> e;
        jobs[i] = {e, s};
    }
    sort(all(jobs)); // 終了時刻の早い順

    ll ans = 0, cur = -1; // cur = 直前に選んだ仕事の終了時刻
    for (auto [e, s] : jobs) {
        if (s >= cur) {
            ans++;
            cur = e;
        }
    }
    cout << ans << endl;
}
```

**計算量**：ソート $O(N \log N)$ + 線形スキャン $O(N)$ → $O(N \log N)$。

## 3-4. 交換論法

貪欲法が正しいかどうかを確認するための思考法。

### 考え方

「貪欲法の選択とは異なる最適解があったとして、貪欲法の選択に近づけても悪化しない」ことを示す。

**区間スケジューリングへの適用**

ある最適解が終了時刻の最も早い仕事 $A$ を含まず、別の仕事 $B$（終了時刻が $A$ 以降）を先頭に選んでいたとする。
$B$ を $A$ に置き換えても、$A$ の終了時刻 ≤ $B$ の終了時刻なので、後続の選択肢は減らない。
→ 終了時刻が早い仕事を優先しても損しない。

### 実装で確認できるチェックリスト

```
1. 「何でソートするか」を決める
2. 貪欲に選んだとき、後に選べる候補は減るか？
   → 減らない（または同じ）なら貪欲法が成り立つ
3. 小さい例で手で確かめる
```

貪欲法が成り立たない例：一般的なナップサック問題（重さが異なるとき価値順ソートは最適でない → DP を使う）。

## まとめ

| パターン | 戦略 | 計算量 |
| --- | --- | --- |
| 価値/コスト最適化 | 基準でソートして先頭から選ぶ | $O(N \log N)$ |
| 区間スケジューリング | 終了時刻の早い順に選ぶ | $O(N \log N)$ |
| 締切スケジューリング | 締切の早い順に処理する | $O(N \log N)$ |

### 貪欲法の思考フロー

```
「毎回何を基準に選ぶか」を決める
        ↓
交換論法で「その選択が後の選択肢を狭めないか」を確認
        ↓
小さい例で手を動かして確かめる
        ↓
ソート + 線形スキャンで実装
```

### DP と迷ったとき

- 選択肢同士が**干渉しない**（選んでも後の候補が変わらない）→ 貪欲法
- 選択の影響が**後に波及する**（残り容量・残りコストなど状態が変わる）→ DP
