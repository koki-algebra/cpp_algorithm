# 第2章：全探索

## 2-1. 全探索の考え方

全探索（ブルートフォース）とは、**考えられる全ての候補を列挙して答えを求める**手法です。

- 実装が素直でバグが少ない
- 計算量が大きいため、制約が小さいときにしか使えない
- まず全探索で考え、TLE なら高速化を検討するという思考フローが重要

### 全探索が使える目安

| 手法 | 計算量 | 使える制約 |
| --- | --- | --- |
| 一重ループ | $O(N)$ | $N \leq 10^8$ |
| 二重ループ | $O(N^2)$ | $N \leq 3000$ |
| ビット全探索 | $O(2^N)$ | $N \leq 20$ |
| 順列全探索 | $O(N! \cdot N)$ | $N \leq 10$ |

## 2-2. ループ全探索

### 一重ループ：条件を満たす要素を探す

```cpp
int main() {
    ll n;
    cin >> n;
    vl a(n);
    rep(i, n) cin >> a[i];

    ll ans = 0;
    rep(i, n) {
        if (a[i] % 2 == 0) ans++;
    }
    cout << ans << endl;
}
```

### 二重ループ：全ペアを調べる

2要素の組み合わせ $(i, j)$（$i < j$）を全列挙する。

```cpp
int main() {
    ll n;
    cin >> n;
    vl a(n);
    rep(i, n) cin >> a[i];

    ll ans = 0;
    rep(i, n) {
        rep2(j, i + 1, n) {
            if (a[i] + a[j] == 0) ans++;
        }
    }
    cout << ans << endl;
}
```

**計算量の確認**：$N = 3000 \Rightarrow N^2 = 9 \times 10^6$（OK）。$N = 10^5 \Rightarrow N^2 = 10^{10}$（TLE）。

## 2-3. ビット全探索

$N$ 個の要素それぞれについて「選ぶ / 選ばない」の $2^N$ 通りの部分集合を全列挙する手法。

### 仕組み

整数 `bit`（$0$ 以上 $2^N$ 未満）の $i$ ビット目が $1$ なら「$i$ 番目の要素を選ぶ」と解釈する。

```
bit = 0b101 (= 5) のとき
  → 0番目と2番目を選ぶ
```

### 基本パターン

```cpp
int main() {
    ll n;
    cin >> n;
    vl a(n);
    rep(i, n) cin >> a[i];

    ll ans = 0;
    rep(bit, 1LL << n) {            // 0 〜 2^n - 1 を全列挙
        ll sum = 0;
        rep(i, n) {
            if (bit & (1LL << i)) { // i ビット目が 1 = i 番目を選ぶ
                sum += a[i];
            }
        }
        if (sum == 0) ans++;
    }
    cout << ans << endl;
}
```

### ビット演算の基本

| 演算 | 記号 | 意味 | 例（`a=5`, `b=3`） |
| --- | --- | --- | --- |
| AND | `&` | 両方 1 なら 1 | `5 & 3` → `1` |
| OR | `\|` | どちらか 1 なら 1 | `5 \| 3` → `7` |
| XOR | `^` | 異なるなら 1 | `5 ^ 3` → `6` |
| 左シフト | `<<` | ビットを左へ動かす | `1LL << 3` → `8` |

**`1LL << i`**：$i$ ビット目だけが $1$ の整数。`int` の範囲を超える可能性があるため `1LL`（`long long`）を使う。

### チェックの定石

「$i$ ビット目が立っているか」の判定は `bit & (1LL << i)` が $0$ でないかどうか。

```cpp
if (bit & (1LL << i)) {
    // i 番目を選んでいる
}
```

## 2-4. 順列全探索

$N$ 個の要素の並び方（$N!$ 通り）を全列挙する手法。STL の `next_permutation` を使う。

### 基本パターン

```cpp
int main() {
    ll n;
    cin >> n;
    vl a(n);
    rep(i, n) cin >> a[i];
    sort(all(a)); // 昇順から始めないと全順列を列挙できない

    ll ans = LLONG_MIN;
    do {
        ll val = 0;
        rep(i, n) val += a[i] * (i + 1); // 並び順に応じた評価値の例
        ans = max(ans, val);
    } while (next_permutation(all(a)));

    cout << ans << endl;
}
```

**注意**：`next_permutation` は現在の並びから「辞書順で次の順列」に進む。昇順ソートした状態から始めないと、全順列を網羅できない。

### N! の感覚

| $N$ | $N!$ |
| --- | --- |
| 8 | 40,320 |
| 10 | 3,628,800 |
| 12 | 479,001,600 |

$N \leq 10$ が実用の上限。

---

## まとめ

| 手法 | いつ使う | 計算量 |
| --- | --- | --- |
| 一重ループ | 各要素を個別に調べる | $O(N)$ |
| 二重ループ | 全ペアを調べる | $O(N^2)$ |
| ビット全探索 | 部分集合を全列挙（$N \leq 20$） | $O(2^N)$ |
| 順列全探索 | 全順列を列挙（$N \leq 10$） | $O(N! \cdot N)$ |

### 全探索の思考フロー

```
何を全列挙するか決める
  → 各要素？   → ループ全探索
  → 部分集合？ → ビット全探索（N ≤ 20）
  → 全順列？   → 順列全探索（N ≤ 10）
        ↓
制約から計算量を確認（TLE しないか？）
        ↓
各候補に対して条件判定 or 評価値計算
```
