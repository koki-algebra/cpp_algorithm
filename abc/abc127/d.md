# ABC127 D - Integer Cards

- 問題: <https://atcoder.jp/contests/abc127/tasks/abc127_d>

## 問題の言い換え

- $N$ 枚のカードがあり、$i$ 枚目には整数 $A_i$ が書かれている。
- $M$ 回の操作を順に行う。操作 $i$ は「好きな $B_i$ 枚のカードを選び、書かれている整数を $C_i$ に書き換える」。
- 全操作終了後、$N$ 枚のカードに書かれた整数の合計を最大化したい。その最大値を求めよ。

## 解法の考え方

各操作でどのカードを選ぶかは完全に自由なので、方針は「値が小さいカードほど積極的に大きい $C_i$ で置き換える」こと。

- 操作を $C_i$ の**降順**に並べ替える。
- カードの値全体を**最小ヒープ**で管理する(初期状態は $A_1, \dots, A_N$)。
- $C_i$ が大きい操作から順に、ヒープの最小値を $B_i$ 回 pop して $C_i$ を push する(「一番小さい値を $C_i$ に書き換える」ことに相当)。
  - ただし `top() >= C_i` になったら、それ以上置き換える意味がないのでその操作は打ち切る。
- 全操作が終わったら、ヒープの中身をすべて合計する。

### なぜ $C_i$ の降順で貪欲にやってよいか

各操作は「好きなカードを選べる」という自由度があるため、実際に操作が行われる時刻の前後関係に関わらず、「どのカードにどの操作の値を残すか」は後から自由に選べる。大きい $C_i$ ほど小さい値を上書きする権利を優先的に与えたほうが、最終的な合計は損をしない(交換しても悪化しない、という交換法則的な議論で正当化できる)。したがって降順に処理して、その時点で一番小さい値から置き換えていくのが最適。

## 計算量

$O((N+M)\log(N+M))$。

| 処理 | 計算量 |
|---|---|
| 最小ヒープ構築(`push` を $N$ 回) | $O(N \log N)$ |
| 操作を $C$ の降順にソート | $O(M \log M)$ |
| メインの二重ループ | $O((N+M) \log N)$ (下記の理由) |
| ヒープから取り出してソート | $O(N \log N)$ |
| 全カードの合計 | $O(N)$ |

### なぜメインループが $O(NM)$ にならないか

`rep(j, b)` があるので一見 $O(\sum B_i)$ 回まわりそうだが、実際はそうならない。

- 操作は $C$ の降順に処理しているので、あるカードの値が一度 $C_i$ に置き換えられると、以降処理される操作の $C_j$ は必ず $C_j \le C_i$。
- そのカードが再びヒープの最小値になったとき、`pq.top() (= C_i) >= C_j` が成り立つため、即座に `break` して二度と置き換えられない。

つまり **ヒープの各要素は生涯で高々1回しか pop\&push されない**ため、pop/push の総回数は操作全体を通して $O(N)$ に抑えられる。空振りの `break` も合わせた内側ループの実行回数の総和は $O(N+M)$ であり、各回 $O(\log N)$ のヒープ操作なので、メインループ全体は $O((N+M)\log N)$ となる。

$N, M \le 10^5$ 程度の制約であれば十分高速に収まる。
