# ABC126 D - Even Relation

- 問題: <https://atcoder.jp/contests/abc126/tasks/abc126_d>
- 参考: <https://drken1215.hatenablog.com/entry/2019/05/19/224500>

## 問題の言い換え

- $N$ 頂点の重み付き木が与えられる。
- 各頂点を白(0)・黒(1)で塗り分ける。
- 条件: **同じ色の任意の2頂点間のパス長は必ず偶数**になること。
- そのような塗り方が必ず存在することが保証されている。

## 解法の考え方

パス長の偶奇は、通る辺の重みの偶奇の総和の偶奇だけで決まる(偶数の重みは総和の偶奇に影響しない)。そこで根(頂点0)から

- 奇数の辺を渡るたびに色を反転
- 偶数の辺を渡っても色はそのまま

というルールで塗っていけば、任意の2頂点間のパス長の偶奇は「その2頂点の色が同じか違うか」と一致する(同色→偶数、異色→奇数)。木なので頂点間の経路は一意に定まり、矛盾なく塗り分けられる。

## なぜそうなるか: ステップを追って確認する

### ステップ1: 距離は「辺の重みの総和」

木では2頂点間の経路は一意なので、距離 = その経路上の辺の重みの単純な足し算。

### ステップ2: 総和の偶奇は「各項の偶奇」だけで決まる

数の総和の偶奇を考えるとき、各項の値そのものは関係なく、各項が偶数か奇数かだけが効く。

- 偶数 + 偶数 = 偶数
- 偶数 + 奇数 = 奇数
- 奇数 + 奇数 = 偶数

つまり偶数の項はいくつ混ざっていても偶奇に影響せず、奇数の項が何個あるか(の偶奇)だけで全体の偶奇が決まる。

### ステップ3: 「根から各頂点までの奇数辺の数(の偶奇)」= 塗る色

DFSで「奇数辺を渡ったら反転、偶数辺なら維持」というルールで色 $c(v)$ を決めていくと、これは厳密に「根から $v$ までの経路上にある奇数の辺の本数を2で割った余り」になっている。ステップ2より、これは「根から $v$ までの距離の偶奇」そのものでもある。

### ステップ4: 具体例で確認

```
        0
      /   \
   (3)     (4)
    1       2
  /   \
(2)   (5)
 3     4
```

辺 0-1 の重み3(奇)、0-2 の重み4(偶)、1-3 の重み2(偶)、1-4 の重み5(奇)。

色を計算すると:

- $c(0) = 0$(開始点)
- $c(1) = 1$(0-1は奇数辺なので反転)
- $c(2) = 0$(0-2は偶数辺なので維持)
- $c(3) = 1$(1-3は偶数辺なので維持)
- $c(4) = 0$(1-4は奇数辺なので反転)

頂点3と頂点4の実際の距離: 経路 3→1→4 で重み $2+5=7$(奇数)。色は $c(3)=1, c(4)=0$ で異なる色 → 「異色なら奇数」と一致。

頂点2と頂点4の距離: 経路 2→0→1→4 で $4+3+5=12$(偶数)。色は $c(2)=0, c(4)=0$ で同じ色 → 「同色なら偶数」と一致。

### ステップ5: なぜ一般に成り立つか

2頂点 $u, v$ の距離は、共通の祖先(LCA)を $m$ とすると

$$
\text{distance}(u, v) = (\text{根から} u \text{までの距離}) + (\text{根から} v \text{までの距離}) - 2 \times (\text{根から} m \text{までの距離})
$$

という式で書ける(根から $m$ までの部分を二重に数えているので引く)。最後の項 $2 \times (\dots)$ は必ず偶数なので、偶奇には一切影響しない。したがって

$$
\text{distance}(u, v) \text{ の偶奇} = c(u) \oplus c(v)
$$

が常に成り立つ。これはどんな2頂点の組み合わせに対しても(親子関係でなくても、遠い枝同士でも)成立する一般的な等式であり、これが「同色→偶数、異色→奇数」という塗り分けが木全体で矛盾なく成立する理由になる。

## 実装方針(反復DFS + visited配列)

再帰DFSでも解けるが、木が一直線(パス状)に近い形だと再帰の深さが $N$(最大 $10^5$)に達し、コールスタックが溢れるリスクがある。ヒープ上に確保される `std::stack` を自前で使う反復DFSならその心配がない。

- スタックには `{頂点, 色}` のペアを積む。
- `vector<bool> visited(n)` で訪問済みかどうかを管理する。
- 計算量は $O(N)$。

```cpp
vl ans(n, 0);
vector<bool> visited(n, false);
stack<pll> st;  // {頂点, 色}
st.push({0, 0});
while (!st.empty()) {
    auto [v, c] = st.top();
    st.pop();
    if (visited[v]) continue;
    visited[v] = true;
    ans[v] = c;

    for (auto& [nv, w] : graph[v]) {
        if (visited[nv]) continue;
        ll nc = (w & 1) ? (1 - c) : c;
        st.push({nv, nc});
    }
}
```
